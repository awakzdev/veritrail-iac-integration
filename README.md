# AWS Free Baseline, Terraform + Terragrunt

A multi-environment AWS baseline designed to be **free-by-default**.

It creates only AWS resources that are normally no-additional-charge by themselves:

- IAM password policy, roles, managed policies, and optional account alias
- VPC, subnets, route tables, network ACLs, Internet Gateway, and security groups
- Optional S3/DynamoDB gateway VPC endpoints, disabled by default

It intentionally does **not** create NAT Gateways, EC2 instances, EIPs, RDS, EKS, load balancers, KMS keys, AWS Config, CloudWatch log groups, Route 53 hosted zones, or S3 buckets.

> Important: AWS can still charge for traffic, free-tier overages, enabled services outside this repo, or future pricing changes. This project is a “no paid resources by default” scaffold, not a legal pricing guarantee.

## What gets created

### IAM baseline

In `live/global/account`:

- Account password policy
- Optional account alias, disabled by default because an account can have only one alias
- Global cost guardrail policy

Per environment:

- `readonly` role with AWS managed `ReadOnlyAccess`
- `security-audit` role with AWS managed `SecurityAudit`
- A reusable explicit-deny cost guardrail policy

The roles trust the current AWS account root by default. That does **not** mean everyone can assume them. A principal still needs `sts:AssumeRole` permission.

### Network baseline

Per environment:

- One VPC
- Two public subnets across the first two available AZs
- One Internet Gateway
- One public route table
- Locked-down default security group
- `no-ingress` security group
- `controlled-ingress` security group, empty by default
- Optional gateway endpoints for S3 and DynamoDB, disabled by default

There are no workloads attached, no NAT, no load balancer, and no private subnets requiring paid egress infrastructure.

## Prerequisites

- Terraform `>= 1.6`
- Terragrunt `>= 0.55`
- AWS CLI authenticated to the target account

Example:

```bash
aws sts get-caller-identity
```

Set a region if you do not want the default:

```bash
export AWS_REGION=eu-west-1
```

## Safety check before apply

Run the static guardrail scanner:

```bash
python3 scripts/verify_no_paid_resources.py
```

This looks for common paid resource types that should not appear in this repository.

## State backend

This repo uses a **local Terraform backend** generated by Terragrunt under:

```text
.state/<env>/<component>/terraform.tfstate
```

This avoids creating an S3 bucket and DynamoDB table, because those can create charges outside the always-free shape.

For a real team setup, move state to S3 + DynamoDB intentionally, but treat that as a paid/operational decision.

## Cost-safe things intentionally avoided

| Avoided resource | Why |
|---|---|
| `aws_nat_gateway` | Hourly + data processing charges |
| EC2 instances | Compute charges outside free tier or after free tier |
| Elastic IPs | Can charge when idle or attached depending on usage |
| RDS / EKS / ELB | Paid services |
| AWS Config | Configuration item and rule evaluation pricing |
| KMS customer managed keys | Monthly key charges |
| CloudWatch logs / alarms | Logs, metrics, and alarms may charge |
| Route 53 hosted zones | Monthly hosted zone charge |
| S3 buckets | Storage/request charges; free tier is account-age and usage dependent |

## Customization

Edit each environment file:

```text
live/dev/_env.hcl
live/staging/_env.hcl
live/prod/_env.hcl
```

Example knobs:

```hcl
locals {
  environment           = "dev"
  name_prefix           = "free-dev"
  vpc_cidr              = "10.10.0.0/16"
  public_subnet_cidrs   = ["10.10.1.0/24", "10.10.2.0/24"]
  allowed_ingress_cidrs = []
  allowed_ingress_ports = []
}
```

To allow inbound HTTPS from your office IP:

```hcl
allowed_ingress_cidrs = ["203.0.113.10/32"]
allowed_ingress_ports = [443]
```

## Notes

- Security groups are not a security product by themselves; they are rules attached to future workloads.
- The explicit-deny cost guardrail policy is helpful, but not exhaustive. AWS has too many billable paths for a single policy to be perfect.
- For production, you would usually add CloudTrail, AWS Config, GuardDuty, centralized state, logging, alerts, and budget alarms. Those are intentionally not included here because the request was free-to-use infrastructure.
